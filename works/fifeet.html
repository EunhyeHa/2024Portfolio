project goal

workflow

W01

- 주제 선정, 경쟁사 리서치
- 주제 분석, 경쟁사 3사 앱 분석 및 SWOT 분석
- 분석 범위 확정, SWOT 재분석 → 개선방법 재정립
- 페르소나 자료조사 및 작성
- 페르소나, UX VISION, 문제점 및 개선방향 정립
- 사용자 시나리오 작성 / 메뉴트리 작성

W02

- 와이어프레임 작업 시작
- 회원정보 와이어프레임 수정 → 공유 → 종합
- 체크리스트 작성 → 프로젝트의 타당성, 니즈 등
- 와이어프레임 + 워크플로우 제작
- 프로토타입 → 기본 구성, 레이아웃 설정 후
- 프로토타입 작업 / 디자인 디테일 수정

W03

- 리뷰 1차 / 폰트 자간, 행간, 아이콘 간격 등 조절
- 디자인 마무리, 2차 리뷰, 디자인 소스 정리
- 프로토 타입 완성 / 최종 발표 준비
- 발표 준비 마무리, 연습
- 최종 발표 / 질의응답

review

    good
  • 사용자 경험을 고려한 인터페이스에 대한 이해도를 높일 수 있었다.
  • 목표로한 접근성과 편리성이 좋은 결과 도출하였다.
  • 팀원들과 소통하고 의견을 조율하는 방법의 이해할 수 있었다.
    bad
  • 자료 조사 과정에서 ‘전시회' 관련 인사이트의 수집의 한계를 느꼈다.
  • 리뉴얼 대상인 ‘아트보다 갤러리'만의 색을 가진 앱으로 발전의 부족함을 느꼈다.
    takeaway
  • 지금까지 진행했던 프로젝트와는 다르게 많은 양의 자료와 소스를 취합하면서 선택과 집중의 중요성을 배웠다.
  • CSS 작성할 때, 중복되는 부분을 줄이고 더 최적화하여 작성할 수 있는 방법을 습득했다.
  • 스크립트 구현시 100% 실행되지 않았지만, 원리를 이해하고 다른방법을 통해 비슷한 동작을 구현해냈다.
